1. Como funciona o ciclo comercial

São dois caminhos para o cliente entrar:

  • Autoatendimento: o cliente acessa /checkout.html, escolhe o módulo e paga. Ao confirmar o pagamento, o sistema ativa e envia o link de ativação automaticamente.
  • Venda assistida (você no controle): você emite a licença pelo painel e envia o link você mesmo (WhatsApp/e-mail). Ideal para negociações e cobrança avulsa.
Módulos: Básico (25 clientes), Intermediário (100), Avançado (150). Recorrente tem 5% de desconto e renova sozinho; avulso bloqueia no vencimento.

2. Emitir licença e enviar para o cliente

1
No painel, clique em “+ Emitir licença” (topo do Painel ou aba Licenças).
2
Preencha nome, e-mail e WhatsApp do cliente, escolha o módulo e o tipo de cobrança.
3
Clique em “Gerar licença e link”. O sistema cria o cliente e mostra três coisas prontas:
  • Link de ativação — botão Copiar.
  • Licença (ex.: DPO-XXXX-XXXX-XXXX) — botão Copiar.
  • Mensagem pronta (com o passo a passo já escrito) — botão Copiar mensagem e, se houver WhatsApp, Enviar no WhatsApp (abre a conversa já com o texto).
Para reenviar depois: aba Licenças → botão “Enviar/Copiar” na linha do cliente. Cada envio fica registrado na Auditoria.

3. O que o cliente faz (ativação)

  1. Abre o link recebido (cai em /ativar com a licença já preenchida).
  2. Cria o usuário dele (e-mail + senha).
  3. Confirma a licença → o módulo é destravado na primeira vez e o acesso fica ativo.

A partir daí ele entra pela tela de login em /.

4. Kill-switch: bloquear / reativar

O controle de acesso é feito no servidor a cada requisição. Você pode agir manualmente:

  • Suspender (aba Licenças → Suspender): corta o acesso imediatamente. Reversível.
  • Reativar: devolve o acesso (após regularização, por exemplo).
  • Revogar: encerra a licença em definitivo (cancela a assinatura do cliente).
Bloqueios por inadimplência acontecem sozinhos (veja item 6). Use a ação manual para casos pontuais.

5. Upgrade de módulo e cota (sem perder dados)

Na aba Assinantes, na linha do cliente:

  • Módulo: troca para outro plano (upgrade/downgrade). A cota de clientes se ajusta automaticamente e nenhum dado do cliente é perdido — o ambiente é o mesmo para todos os módulos.
  • Cota: define um limite sob medida (ex.: um cliente com 200 clientes). Vazio = usa a cota do módulo.

6. Cobrança automática e vencimentos

Todo dia o sistema:

  1. Avisa o cliente 3 dias antes do vencimento.
  2. No vencimento, entra em carência (padrão 5 dias).
  3. Passada a carência, bloqueia o acesso (kill-switch) e suspende a licença.
  4. Quando o pagamento é confirmado (webhook do gateway), reativa automaticamente.

No Painel, o bloco “Vencimentos & atrasos” mostra quem está vencendo/atrasado, com botão Cobrar (abre o WhatsApp com a mensagem pronta).

7. Financeiro e NFS-e

Aba Financeiro: lista os pagamentos. Para pagamentos confirmados, o botão “Emitir NFS-e” dispara a nota (se a integração Focus NFe estiver configurada).

8. Seus clientes de consultoria

Aba Meus clientes: seu ambiente próprio, ilimitado, para cadastrar os clientes das suas consultorias — separado dos assinantes da plataforma.

9. Auditoria e versionamento

Aba Auditoria: registro imutável de todos os eventos de licença (emitida, enviada, ativada, suspensa, reativada, revogada, upgrade/downgrade, mudança de cota). Cada licença tem um número de versão que incrementa a cada mudança de estado — rastreabilidade total para fins legais e de conformidade.

10. Checklist de venda (resumo)

  1. Painel → + Emitir licença → preencher dados + módulo + cobrança.
  2. Copiar mensagem (ou Enviar no WhatsApp) e mandar para o cliente.
  3. Cliente ativa pelo link (cria usuário + confirma licença).
  4. Acompanhar pagamento no Financeiro; emitir NFS-e.
  5. Monitorar Vencimentos; cobrar com 1 clique se necessário.
  6. Upgrade/cota conforme o cliente cresce — sem perder dados.
